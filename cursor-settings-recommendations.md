# Cursor Settings Recommendations

> This file is a guide, not auto-applied config. Review each recommendation and apply
> what makes sense for your project in Cursor Settings or your user/workspace settings.json.
> Generated by cursor-setup skill. Delete this file after applying your choices.

## High-impact settings (apply these first)

### 1. Enable Workspace Trust

Cursor ships with Workspace Trust disabled by default, which means untrusted repos can
auto-execute tasks on open. Enable it:

```json
{
  "security.workspace.trust.enabled": true
}
```

Tradeoff: you'll need to mark your project folders as trusted the first time you open them.

### 2. Enable Privacy Mode

If you work on proprietary, client, or regulated code, enable Privacy Mode in
Cursor Settings > Privacy. This affects whether code/prompts may be retained or used
for training when privacy mode is off.

Even with your own API key, requests still transit Cursor infrastructure for final prompt
building. Privacy Mode is a separate layer on top of API key usage.

### 3. Disable telemetry (optional)

```json
{
  "telemetry.telemetryLevel": "off",
  "workbench.enableExperiments": false
}
```

Tradeoff: disabling experiments may delay access to A/B-tested feature rollouts.

## Agent workflow settings

### 4. Plan Mode toggle

Use `Shift+Tab` in the agent input to toggle Plan Mode. Use Plan Mode for:
- Multi-file refactors
- New features crossing boundaries
- Infrastructure changes
- Migrations

Use direct execution for narrow, well-scoped changes.

### 5. Context and token management

Cursor's dynamic context discovery (January 2026+) means the old "every MCP tool bloats
every prompt" model is no longer accurate. But you still benefit from:

- Keeping `.cursorindexingignore` well-maintained (reduces semantic search noise)
- Starting fresh chats for new tasks (long chats dilute rule compliance)
- Writing long tool outputs to files and letting the agent read them (reduces context cost)

## Tab completion settings

### 6. Reclaim the Tab key (highest-ROI keybinding change)

If Tab autocomplete fights with indentation, rebind "Accept Cursor Tab Suggestion" to a
different key (e.g., `Ctrl+.` or `Alt+Enter`). This is the single most recommended
keybinding change in the Cursor community.

### 7. Enable Partial Accepts

Turn on partial (word-by-word) acceptance for tighter control over Tab suggestions.

## Network settings

### 8. HTTP Compatibility Mode

Many "Cursor feels slow" reports are fixed by switching from HTTP/1.0 fallback to HTTP/2
in Cursor Settings > Network.

- Default to HTTP/2 when stable
- Switch to HTTP/1.1 if you hit connection failures on VPN/corporate proxy
- Treat this as a standard first debug step for latency issues

## Model configuration

### Recommended routing for agentic work

- **Default agent model:** Composer 2 (Fast) for high-volume, everyday coding
- **Escalation for hard problems:** Claude Opus 4.6 for deep planning and cross-file reasoning
- **Cost step-down:** Claude Sonnet 4.6 for strong reasoning at lower cost

### Subagent model strategy

- Built-in subagents (Explore, Bash, Browser) use cheaper/faster models by default and
  cannot currently be overridden
- Custom subagents: use `model: fast` for search/verification workers, `model: inherit`
  only for workers that need the parent's reasoning depth
- On legacy request-based plans, subagents may fall back to Composer defaults unless
  Max Mode is enabled

## User Rules recommendation

Put these in Cursor Settings > Rules (User Rules). They apply across all projects and
supplement the project-level rules in `.cursor/rules/`:

```
- Proceed on clear, reversible, low-risk edits; ask before destructive or external-side-effect actions.
- Be concise. End with: files changed, validation run, remaining risks.
- Do not commit or push unless I explicitly ask.
```

Keep User Rules short and personal. Put repo-specific facts in `.cursor/rules/` and `AGENTS.md`.
